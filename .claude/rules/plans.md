## Rules:

1. Before start planning, ask user questions if you think you need more information to make a better plan.
2. Each task must come with a [x] to track its completion.
3. Each plan must include testing cases
4. Output the plans at folder plans/<feature>, with file name convention <feature>.<scope: backend or frontend>.md
5. For the backend scope, the plan must include all api endpoints added, single line only with method, path, and a brief description.
